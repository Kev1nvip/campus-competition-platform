## 六、分工说明

```
关于使用AI辅助编码：
  鼓励所有成员使用AI工具（GitHub Copilot、Cursor等）辅助开发
  AI生成的代码必须理解后再提交，能向队友解释自己写的代码
  答辩时老师可能提问任何模块，每人需要熟悉自己负责的部分

关于后端队友C：
  SpringBoot基础薄弱没关系
  负责的模块全部是标准CRUD逻辑
  可以参考负责人写好的核心模块代码风格
  遇到问题先问AI，再问负责人

关于环境：
  所有人只需要拉取代码，运行docker compose up -d
  不需要自己配置中间件
  前端本地开发：npm install && npm run dev
  后端本地开发：VS Code Spring Boot Dashboard启动
```
